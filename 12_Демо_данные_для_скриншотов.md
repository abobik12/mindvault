## Сообщения в чате

Перед скриншотом чата отправить несколько сообщений. Часть сообщений создаст объекты, часть покажет обычный вопрос к ассистенту.

### Сообщение 1

```text
заметка Проверить структуру главы 3 перед переносом текста в Word
```

Ожидаемый результат: создается заметка.

### Сообщение 2

```text
список Подготовка к защите: проверить презентацию, открыть приложение, выполнить тесты
```

Ожидаемый результат: создается список.

### Сообщение 3

```text
напоминание завтра в 10:00 проверить скриншоты интерфейса
```

Ожидаемый результат: создается напоминание, если парсер корректно распознает дату и время.

### Сообщение 4

```text
Какие материалы по ВКР у меня сохранены?
```

Ожидаемый результат: ассистент отвечает с учетом заметок, файлов, списков и напоминаний, если AI API доступен. Если внешний AI API недоступен, для скриншота лучше использовать сообщения с локальным созданием заметок, списков и напоминаний, чтобы не показывать ошибку внешнего сервиса.

## Команды для терминальных скриншотов

Запуск Docker Compose:

```bash
docker compose up -d --build
```

Проверка контейнеров:

```bash
docker compose ps
```

Backend tests:

```bash
corepack pnpm --filter @workspace/api-server test
```

Backend typecheck:

```bash
corepack pnpm --filter @workspace/api-server run typecheck
```

Frontend typecheck:

```bash
corepack pnpm --filter @workspace/mindvault run typecheck
```

Общая TypeScript-проверка workspace:

```bash
corepack pnpm run typecheck
```

## Что не показывать и не утверждать

На скриншотах и в подписях не нужно показывать или описывать как реализованные:

- push-уведомления;
- OCR;
- vector search или semantic search;
- production HTTPS/nginx;
- полноценное мобильное приложение;
- интеграции с Telegram или Google Calendar;
- роли администратора;
- оплату или подписки;
- совместную работу;
- end-to-end encryption.
