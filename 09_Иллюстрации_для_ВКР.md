# Иллюстрации для ВКР

Все схемы подготовлены в формате SVG и расположены в папке `vkr_assets/`. Файлы имеют светлый фон, темный текст и строгий технический стиль. Реальные скриншоты интерфейса для рисунков 7-15 в этом этапе не создавались.

## Рис. 1. Общая архитектура приложения MindVault

- Файл: `vkr_assets/fig01_architecture.svg`.
- Что изображено: пользовательский браузер, React + Vite frontend, Express API backend, PostgreSQL, внешний OpenAI/OpenRouter-compatible API и модуль формирования пользовательского контекста.
- Где вставить: вместо placeholder `[ВСТАВИТЬ РИС. 1. Общая архитектура приложения MindVault]` в разделе 2.3.
- Подпись: `Рис. 1. Общая архитектура приложения MindVault`.

## Рис. 2. Диаграмма развертывания приложения MindVault

- Что изображено: локальный запуск через Docker Compose: контейнеры `web`, `api`, `db`, volume `pg_data`, порты 18174, 8080 и 5432, внешний AI API.
- Где вставить: вместо placeholder `[ВСТАВИТЬ РИС. 2. Диаграмма развертывания приложения MindVault]` в разделе 2.3.
- Подпись: `Рис. 2. Диаграмма развертывания приложения MindVault`.
- Примечание: схема не показывает production HTTPS/nginx, так как это не реализовано в проекте.

## Что вставить вручную в Word

- Вставить каждый SVG вместо соответствующего placeholder в тексте.
- Оставить подписи рисунков такими же, как в вычитанном Markdown-файле.
- После вставки проверить масштаб: текст в схемах должен читаться на листе А4.
- Для рисунков 7-15 отдельно вставить реальные скриншоты интерфейса, Docker-запуска и результатов тестов.
