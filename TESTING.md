# TESTING.md

Чек-лист финальной проверки MindVault. Используйте его перед деплоем и после крупных правок логики ассистента.

## Запуск проекта

1. Остановить старые контейнеры:
   ```bash
   docker compose down
   ```
2. Собрать и запустить проект:
   ```bash
   docker compose up -d --build
   ```
3. Проверить состояние:
   ```bash
   docker compose ps
   docker compose logs api
   docker compose logs web
   ```
4. Открыть сайт: http://localhost:18174

Ожидаемо: `api`, `web` и `db` запущены, в логах нет ошибок старта.

## Сборка

Проверить доступные команды проекта:

```bash
pnpm --filter @mindvault/api-server test
pnpm --filter @mindvault/api-server build
pnpm --filter mindvault build
```

Если команда падает, сначала смотрите точную ошибку сборки или TypeScript, затем проверяйте последние изменения.

## Тесты

Автоматические тесты должны покрывать разбор команд:

- `заметка ...`
- `напоминание ...`
- `список ...`
- обычные вопросы без автосохранения
- спорные случаи с предложением действия
- пустой и очень длинный ввод
- HTML/JS в тексте как обычный текст

Команда:

```bash
pnpm --filter @mindvault/api-server test
```

## Проверка чата

1. Отправить `как лучше изучать Java?`.
   Ожидаемо: ассистент отвечает как на обычный вопрос, заметка/список/напоминание не создаются.
2. Отправить пустое сообщение или пробелы.
   Ожидаемо: приложение не падает, сообщение не создается.
3. Быстро нажать Enter несколько раз.
   Ожидаемо: нет дублей и падений интерфейса.
4. Проверить кнопку прокрутки вниз.
   Ожидаемо: появляется только когда пользователь не внизу, не перекрывается полем ввода.

## Проверка заметок

1. Через чат: `заметка изучить Java`.
   Ожидаемо: создана заметка с текстом `изучить Java`, ИИ для этого не нужен.
2. Через чат: `заметка`.
   Ожидаемо: заметка не создается, ассистент просит ввести текст.
3. Через чат: `заметка список напоминание`.
   Ожидаемо: создается именно заметка `список напоминание`.
4. Открыть страницу "Заметки".
   Ожидаемо: созданные заметки отображаются после обновления страницы.
5. Ввести `<script>alert(1)</script>` как текст заметки.
   Ожидаемо: текст отображается как текст, код не выполняется.

## Проверка напоминаний

1. Через чат: `напоминание завтра в 10:00 стрижка`.
   Ожидаемо: создается напоминание на завтра в 10:00 с текстом `стрижка`.
2. Через чат: `напоминание позвонить преподавателю`.
   Ожидаемо: напоминание не создается, ассистент просит уточнить дату и время.
3. Через чат: `напоминание завтра`.
   Ожидаемо: напоминание не создается, ассистент просит уточнить недостающие данные.
4. Через чат: `напоминание 32 января купить хлеб`.
   Ожидаемо: напоминание не создается с некорректной датой.
5. Открыть страницу "Напоминания".
   Ожидаемо: реальные напоминания отображаются, время не сдвинуто.

## Проверка списков

1. Через чат: `список хлеб, молоко, яйца`.
   Ожидаемо: создан список из трех пунктов.
2. Через чат: `список покупки: хлеб, молоко, яйца`.
   Ожидаемо: создан список "Покупки" с тремя пунктами.
3. Через чат: `список`.
   Ожидаемо: список не создается, ассистент просит ввести пункты.
4. Открыть страницу "Списки" и нажать "Создать список".
   Ожидаемо: можно ввести название и пункты через запятую или переносы строк.
5. Отметить чекбокс пункта и обновить страницу.
   Ожидаемо: состояние чекбокса сохранилось.
6. Проверить длинный пункт списка.
   Ожидаемо: карточка не ломается, текст переносится.

## Проверка спорных вводов

1. `завтра в 10 стрижка`.
   Ожидаемо: ничего не создается автоматически, появляется предложение создать напоминание.
2. Нажать "Создать напоминание".
   Ожидаемо: создается одно напоминание, двойной клик не создает дубль.
3. `хлеб, молоко, яйца`.
   Ожидаемо: ничего не создается автоматически, появляется предложение создать список.
4. Нажать "Создать список".
   Ожидаемо: создается один список, двойной клик не создает дубль.
5. Нажать "Сохранить как заметку".
   Ожидаемо: создается заметка из исходного текста.
6. Нажать "Нет".
   Ожидаемо: предложение исчезает, данные не создаются.
7. `создай заметку изучить Java`.
   Ожидаемо: автосохранения нет, потому что команда не начинается со слова `заметка`.

## Проверка файлов

1. Загрузить TXT/MD/CSV/JSON.
   Ожидаемо: файл отображается, текст извлечен и доступен ассистенту.
2. Загрузить PDF/DOCX.
   Ожидаемо: если текст извлечен, ассистент использует его; если нет, честно сообщает об ограничении.
3. Загрузить неподдерживаемый бинарный файл.
   Ожидаемо: файл сохраняется, мусорный бинарный текст не показывается.
4. Проверить длинное имя файла.
   Ожидаемо: карточка не ломает сетку.

## Проверка мобильной версии

Проверить ширины: `320px`, `375px`, `390px`, `430px`, `768px`.

Обязательно проверить:

- чат и нижнее поле ввода;
- кнопку прокрутки вниз;
- страницу списков и окно создания списка;
- карточки заметок, напоминаний, списков и файлов;
- боковое меню;
- длинные названия и длинный текст.

Ожидаемо: нет горизонтального скролла, кнопки нажимаются, модальные окна помещаются на экран.

## Что делать, если что-то сломалось

1. Проверить логи:
   ```bash
   docker compose logs api
   docker compose logs web
   ```
2. Проверить консоль браузера.
3. Повторить сценарий на свежей странице после reload.
4. Проверить, не создались ли дубли в данных.
5. Запустить тесты command-parser.
6. Если ошибка только в одном сценарии, чинить минимально в соответствующем модуле, не переписывая общий flow.
